Asset Guidelines

Guidelines for using images, illustrations, and 3D assets throughout the platform.

Use WebP Format

All images should be in WebP format for optimal performance. Convert PNG/JPG to WebP with quality 80-90%.

Maximum Width 400px

3D illustrations and icons should have a maximum width of 400px to ensure fast loading and consistent sizing.

Asset Location

Store all assets in frontend/assets/images/. Use subfolders like /3d/ for 3D assets, /badges/ for badges.

Accessibility

Always include descriptive alt attributes for images. Use meaningful descriptions for screen readers.

Image Compression

Compress all images before committing. Use tools like TinyPNG or ImageOptim to reduce file size without quality loss.

Accessibility Guidelines

EcoLife follows accessibility best practices to ensure all users can interact with the platform effectively.

  • Maintain proper color contrast between text and background.
  • Use semantic HTML elements for better screen reader support.
  • Always include descriptive alt text for images.
  • Ensure buttons and links are keyboard accessible.
  • Avoid relying only on color to communicate meaning.

Layout Guidelines

All pages follow a consistent layout structure to maintain visual consistency across the EcoLife platform.

  • Use a maximum container width of 1200px.
  • Maintain consistent spacing between sections.
  • Use card components to group related content.
  • Follow the grid system for responsive layouts.
  • Ensure layouts remain responsive across devices.

Component States

Interactive components support multiple visual states to provide clear feedback to users.

  • Default: Normal appearance of the component.
  • Hover: Indicates the element is interactive.
  • Active: Shows when the element is being clicked.
  • Focus: Used for keyboard navigation accessibility.
  • Disabled: Indicates the action is unavailable.

Responsive Design

EcoLife components are designed to work across multiple screen sizes and devices.

  • Use flexible grid layouts.
  • Ensure cards stack properly on small screens.
  • Maintain readable font sizes on mobile devices.
  • Use media queries to adapt layout.
  • Avoid fixed-width containers.

Animations & Transitions

Subtle animations enhance user experience while maintaining performance.

  • Use smooth transitions for hover interactions.
  • Avoid long or distracting animations.
  • Prefer CSS transitions over heavy JavaScript animations.
  • Keep animations under 300ms for responsiveness.

Contribution Guidelines

When contributing UI components to EcoLife, follow these guidelines to maintain consistency.

  • Use existing CSS variables and design tokens.
  • Follow the established component naming conventions.
  • Ensure new components are responsive.
  • Maintain accessibility standards.
  • Test components across multiple browsers.
